# 第 2 章练习

## 基础练习

1. 创建一个用户列表，字段包含 `id`、`name`、`role`、`enabled`、`score`。
2. 用 `v-for` 渲染列表，并为每一行添加稳定 `key`。
3. 使用 `v-if` 展示空状态：没有用户时显示“暂无数据”。
4. 使用 `v-show` 控制高级筛选区域的显示隐藏。
5. 使用 `:class` 根据 `enabled` 显示不同状态样式。

## 计算属性练习

1. 根据关键字过滤用户列表。
2. 根据分数对用户列表降序排序。
3. 统计启用用户数量。
4. 生成一个页面标题，例如“用户管理（启用 5 人）”。

## 监听练习

1. 监听搜索关键字变化，打印新值和旧值。
2. 给搜索监听加 `immediate: true`。
3. 监听整个表单对象，观察 `deep: true` 的效果。

## 表单练习

1. 创建新增用户表单。
2. 使用 `v-model.trim` 绑定用户名。
3. 使用 `v-model.number` 绑定年龄。
4. 提交时检查用户名是否为空。
5. 新增成功后清空表单。

## 综合练习

实现一个“用户管理小页面”：

- 支持关键字搜索。
- 支持按角色筛选。
- 支持按分数排序。
- 支持新增用户。
- 支持启用/禁用用户。
- 支持删除用户。
- 删除按钮要使用 `.stop` 阻止行点击事件。

## 面试口述题

1. 为什么 computed 适合列表过滤，而 watch 不适合？
2. 为什么 `v-html` 可能造成安全问题？
3. 在后台管理系统里，哪些切换适合 `v-if`，哪些适合 `v-show`？
4. 表单提交时，前端校验和后端校验分别负责什么？
